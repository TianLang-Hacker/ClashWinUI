# 🚀 ClashWinUI

ClashWinUI 是一款基于 WinUI 3 构建的现代化本地网络流量管理与路由控制台。本项目采用前、后端彻底解耦的架构设计，允许用户自定义加载第三方网络处理核心（如 Mihomo），为您提供透明、高效且安全的网络规则配置体验。

[核心特性](#-核心特性) • [安装指南](#-安装指南) • [本地化支持](#-本地化支持) • [开发者指南](#-开发者指南)

---

## ✨ 核心特性

* **网络流量监控**：实时分析与接管本地网络请求，提供直观的流量流向视图。
* **本地网络流分发**：支持构建本地化的网络转发规则，精准控制各个应用的数据通路。
* **规则路由策略**：基于 YAML 配置的强大路由引擎，实现灵活的分流与网络节点管理。
* **现代化原生体验**：完全遵循 Windows 11 的 Fluent Design 设计规范，深度支持 Mica/Acrylic 材质与系统深色模式无缝切换。
* **高度自由的内核管理**：纯前端架构，支持用户自定义引入及切换外部网络核心可执行文件，实现真正的“配置与界面”分离。

## 📦 安装指南

如果您在 Release 页面下载的 `.msix` 安装包由于**本地自签名证书**的原因提示无法直接安装，请按照以下步骤导入信任证书即可解决：

1. **提取证书信息**：右键点击下载的 `.msix` 安装包，选择 **“属性”**。
2. **查看签名**：切换到 **“数字签名”** 选项卡，在列表中选中签名者（如“嵌入式签名”），点击 **“详细信息”**。
3. **安装证书**：在弹出的窗口中点击 **“查看证书”**，随后点击 **“安装证书”**。
4. **选择存储位置**：在证书导入向导中，将存储位置选择为 **“本地计算机”**，点击“下一步”。
5. **指定证书目录**：选择 **“将所有的证书都放入下列存储”**，点击“浏览”，选择 **“受信任的根证书颁发机构”**。
6. **完成安装**：点击“下一步”直到“完成”。系统提示导入成功后，即可双击 `.msix` 文件正常安装应用。

## 🛠️ 开发者指南

欢迎参与项目的开发与维护！请确保您的开发环境满足以下基础要求：

* **IDE 环境**: Visual Studio 2026 或更高版本
* **必需的工作负荷与 SDK**:
    * .NET 桌面开发 (.NET Desktop Development)
    * WinUI 应用程序开发 (WinUI)
    * Windows 11 SDK (10.0.26100.7705)
    * Windows 11 SDK (10.0.22621.0)
    * Windows 通用 C 运行时 (Universal C Runtime)

1. **将项目 Clone 到本地**

```bash
git clone https://github.com/TianLang-Hacker/ClashWinUI.git
```

注：中国大陆地区用户若遇到 Clone 速度过慢的问题，可使用以下在线代理加速：   

```bash
git clone https://gh-proxy.org/https://github.com/TianLang-Hacker/ClashWinUI.git
```

---

2. **编译项目**

你可以使用 Visual Studio 2026 打开 ClashWinUI.sln 解决方案文件后按 F5 启动编译，或者直接使用命令行进行编译：

X86_64：

```bash
dotnet build -c Release -p:Platform=x64
```

ARM64：

```bash
dotnet build -c Release -p:Platform=arm64
```
